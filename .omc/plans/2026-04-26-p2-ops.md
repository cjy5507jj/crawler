## Step 1 — P2.10: crawl_runs 운영 로그 (2h)

**파일**: `sql/migration_005_crawl_runs.sql`, `src/services/run_log.py` (신규), `scripts/run_all.py`

1. `sql/migration_005_crawl_runs.sql`:
   ```sql
   create table if not exists crawl_runs (
     id uuid primary key default gen_random_uuid(),
     started_at timestamptz not null default now(),
     finished_at timestamptz,
     trigger_source text not null default 'manual',  -- manual | launchd | cron
     args jsonb not null default '{}'::jsonb,
     summary jsonb not null default '{}'::jsonb,    -- {category: {source: counts}}
     status text not null default 'running',         -- running | completed | failed
     error text
   );
   create index if not exists idx_crawl_runs_started_at on crawl_runs(started_at desc);
   ```
2. `src/services/run_log.py` 신규 모듈:
   - `start_run(db, *, trigger_source, args) -> run_id`: insert 후 id 반환
   - `record_category(run_id, category, summary_dict)`: 누적 summary jsonb append
   - `finish_run(db, run_id, *, status='completed', error=None)`: finished_at + status update
3. `scripts/run_all.py`:
   - main 진입에서 `start_run` 호출, 종료 시 `finish_run`
   - try/except로 finally 블록에서 finish_run(status='failed', error=str(e)) 보장
   - per-category summary 누적

**Acceptance**: 실행할 때마다 `crawl_runs`에 1 row. summary jsonb에 카테고리별 매칭 카운트 보임.

## Step 2 — P2.9: 변화 감지 알림 (2h)

**파일**: `src/services/alerts.py` (신규), `scripts/run_all.py`

1. `src/services/alerts.py`:
   - `notify(message: str, *, level='info')` 단일 함수
   - 환경변수 우선순위: `SLACK_WEBHOOK_URL` → `DISCORD_WEBHOOK_URL` → stdout fallback
   - httpx POST `{"text": message}` (Slack 호환), Discord는 `{"content": message}`
   - 실패해도 raise 안 함 (alerting이 파이프라인 자체를 중단시키면 안 됨)
2. `scripts/run_all.py` 종료 직전에 anomaly 감지:
   - 마지막 `crawl_runs.summary`와 비교 (직전 1회) — total products / matched count
   - thresholds:
     - 다나와 product count가 직전 대비 50% 미만이면 alert
     - 매칭 카운트가 직전 대비 50% 미만이면 alert
     - 풀 실행 자체가 0 매칭이면 alert
   - alert 메시지 포맷: `🚨 [pc-parts-crawler] {anomaly}: {detail}`
3. 첫 실행(직전 row 없음)이면 baseline만 기록하고 alert 없음.

**Acceptance**: alert 함수 unit test (Slack/Discord/stdout 분기). anomaly 감지 unit test.

## Step 3 — P2.8: launchd 스케줄러 (1h)

**파일**: `ops/com.pcpartscrawler.daily.plist`, `ops/com.pcpartscrawler.daangn-weekly.plist`, `ops/com.pcpartscrawler.vocab-weekly.plist`, `ops/README.md`

1. macOS launchd plist 3개 (cron 보다 안정적):
   - daily 03:00 — `run_all.py --skip-sources daangn --danawa-pages 0`
   - 일요일 04:00 — `run_all.py --danawa-pages 0` (Daangn 포함)
   - 월요일 05:00 — `discover_vocab.py`
   - 각 plist의 `EnvironmentVariables` 블록에 PATH + (optional) SUPABASE_*
   - StandardOutPath / StandardErrorPath → `~/Library/Logs/pc-parts-crawler/{name}.log`
2. `ops/README.md`:
   - 설치 방법: `launchctl bootstrap gui/$UID ~/Library/LaunchAgents/<plist>` (또는 ~/Library/LaunchAgents 심볼릭 링크 + load)
   - 제거: `launchctl bootout`
   - 로그 확인: `tail -f ~/Library/Logs/pc-parts-crawler/...`
   - cron 사용자용 alternate crontab 예시도 포함
3. `trigger_source='launchd'`을 plist에 환경변수로 노출:
   ```xml
   <key>EnvironmentVariables</key>
   <dict>
     <key>CRAWL_TRIGGER_SOURCE</key>
     <string>launchd</string>
   </dict>
   ```
   `run_all.py`에서 `os.environ.get("CRAWL_TRIGGER_SOURCE","manual")`로 사용.

**Acceptance**: plist 파일들 plutil 검증 통과. README 명령어로 설치 가능. 자동 실행 시 crawl_runs.trigger_source='launchd'.
