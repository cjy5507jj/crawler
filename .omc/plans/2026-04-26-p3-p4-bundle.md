## Step 2 — P3.11: 시세 시계열 (2~3h)

**파일**: `sql/migration_006_market_stats_history.sql`, `src/services/aggregate.py`, `scripts/aggregate_stats.py`, `tests/test_aggregate.py`

기존 `product_market_stats`는 latest snapshot 1개만. 1주/4주 트렌드(상승/하락)를 위해 history 테이블에 매 집계마다 스냅샷.

1. `sql/migration_006_market_stats_history.sql`:
   ```sql
   create table if not exists product_market_stats_history (
     id uuid primary key default gen_random_uuid(),
     product_id uuid references products(id) on delete cascade,
     category text not null,
     captured_at timestamptz not null default now(),
     window_days integer not null,
     used_count integer not null default 0,
     used_min integer,
     used_max integer,
     used_median integer,
     used_mean integer,
     new_price integer,
     used_to_new_ratio numeric
   );
   create index if not exists idx_pmsh_product_captured
     on product_market_stats_history(product_id, captured_at desc);
   create index if not exists idx_pmsh_captured
     on product_market_stats_history(captured_at desc);
   ```

2. `src/services/aggregate.py`:
   - `aggregate_market_stats` 끝에 `_bulk_insert_history(db, stats_list, captured_at)` 호출 (insert append, upsert 아님)
   - 새 함수 `compute_trend(history_rows, *, window_days)`:
     - 입력: 단일 product의 history rows (시간순 desc)
     - 출력: `{"trend_pct": float | None, "direction": "up"|"down"|"flat"|None}`
     - window_days 일 전과 가장 가까운 row의 `used_median`을 baseline으로, 현재 row의 `used_median`과 비율 계산
     - baseline이 0/None이거나 history가 부족하면 None 반환
   - 새 함수 `aggregate_trends(db)` (별도 호출):
     - 모든 product에 대해 history 조회 → trend_pct 계산 → `product_market_stats.trend_7d_pct`, `trend_28d_pct` 업데이트
     - 또는 단순화: aggregate_market_stats 직후 호출하면 한 번에 처리
   - `product_market_stats`에 두 컬럼 추가: `trend_7d_pct`, `trend_28d_pct` (nullable). DDL은 migration_006에 같이 포함:
     ```sql
     alter table product_market_stats
       add column if not exists trend_7d_pct numeric,
       add column if not exists trend_28d_pct numeric;
     ```

3. 단위 테스트 `tests/test_aggregate.py`:
   - `test_compute_trend_no_baseline` → None
   - `test_compute_trend_up_5pct`
   - `test_compute_trend_down_10pct`
   - `test_compute_trend_flat_within_2pct` → "flat"

4. `scripts/aggregate_stats.py`에 `--with-trends` 플래그 추가 (기본 true) — history insert + trend update.

**Acceptance**: aggregate 1회 후 `product_market_stats_history`에 row 누적, `product_market_stats.trend_7d_pct/trend_28d_pct` 채워짐 (history 1회만 있으면 None).
