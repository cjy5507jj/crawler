# SECONDHAND_RESEARCH.md

## 목적
다나와를 **기준 상품 마스터(canonical master)** 로 두고, 중고 시세 소스의 게시글/가격을 여기에 매핑하기 위한 조사 문서다.

핵심 목표:
- 다나와 카테고리 / 브랜드 / 모델을 기준축으로 유지
- 중고 소스별로 수집 가능한 가격 데이터를 표준화
- 신품가 vs 중고 최근가/평균가/최저가 비교 기반 마련

---

## 공통 매핑 전략

### 1. 다나와를 기준 상품 마스터로 사용
필수 필드:
- category
- brand
- model_name
- normalized_model
- danawa_product_id
- danawa_url
- current_new_price

### 2. 중고 소스에서 수집할 최소 필드
- source
- listing_id
- title
- price_raw
- price
- seller/location (있으면)
- created_at / crawled_at
- listing_url
- status (판매중/판매완료/알 수 없음)

### 3. 매칭 레이어
중고 게시글을 다나와 상품에 붙일 때 아래 순서로 점수화:
1. 카테고리 일치
2. 브랜드 일치
3. 핵심 모델 토큰 일치 (예: 5600X, RTX 3070, B650M)
4. 불필요 키워드 제거 후 정규화 문자열 비교
5. 제외 규칙 적용 (삽니다, 교환, 고장, 부품용, 예약중 등)

---

## 소스별 조사

## 1) 쿨앤조이

### 접근 방식
- 웹 게시판 HTML 파싱 가능성 검토
- 카테고리별 장터 게시판 URL 구조 파악 필요
- 로그인 없이 게시글 목록/제목/가격 일부 노출 여부 확인 필요

### 수집 접근
- 게시판 목록 페이지 크롤링
- 제목 기반 가격/모델 추출
- 상세 페이지 필요 시 본문/판매완료 여부 확인

### 주요 blocker
- 게시판 구조 변경 가능성
- 글 양식이 자유로워 가격/상태 추출이 불안정할 수 있음
- 과도한 요청 시 차단 가능성

### 데이터 품질 이슈
- 제목 형식이 일정하지 않을 수 있음
- 쿨앤조이 장터는 고급 사용자 비중이 높아 모델 정확도는 좋을 가능성
- 판매완료/예약중/교환 등의 표현이 다양함

### 추천 우선순위
- **상**
- PC 부품 커뮤니티 특성상 모델 정확도가 높아서 다나와 매핑에 유리

---

## 2) 퀘이사존

### 접근 방식
- 장터/중고 게시판 목록 페이지 조사 필요
- HTML 크롤링 중심 접근 예상
- 게시글 본문/댓글에 상태 변화가 들어갈 수 있음

### 수집 접근
- 목록 페이지 제목/가격/작성시각 수집
- 상세 페이지에서 판매완료 여부 보강
- 카테고리별 게시판 분리가 되어 있으면 가장 좋음

### 주요 blocker
- 커뮤니티 구조 변경 가능성
- 일부 정보가 로그인 사용자 기준일 수 있음
- anti-bot 가능성은 중간 수준 예상

### 데이터 품질 이슈
- 하드웨어 커뮤니티라 모델명 품질은 좋을 가능성 큼
- 가격 형식은 자유 텍스트일 수 있음
- 번들/교환/택포 여부 등 부가 조건 많음

### 추천 우선순위
- **상**
- 쿨앤조이와 함께 1차 중고시세 소스로 적합

---

## 3) 당근마켓

### 접근 방식
- 공개 웹 검색 결과 또는 지역 기반 페이지 확인 필요
- 실제 데이터 수집은 지역/동네 제약이 큼
- SPA/동적 로딩 가능성이 높아 Playwright 계열 필요 가능성 큼

### 수집 접근
- 지역별 검색 키워드 수집
- 다나와 기준 상품명에서 검색어 생성 후 검색 결과 파싱
- 지역 편차를 반영해야 함

### 주요 blocker
- 지역 제한
- 동적 렌더링
- anti-bot/비공개 API/접근 제약 가능성 큼

### 데이터 품질 이슈
- 거래량은 많지만 모델명 품질이 낮을 수 있음
- 같은 상품도 제목이 매우 다양함
- 지역 편차 때문에 전국 평균치 해석이 어렵다

### 추천 우선순위
- **중**
- 중요 소스지만 구현 난이도가 높아 커뮤니티 장터 다음 단계가 적절

---

## 4) 번개장터

### 접근 방식
- 검색 페이지 기반 HTML/JSON 추적 가능성 조사
- 공개 검색 결과가 있다면 검색형 크롤러로 접근 가능

### 수집 접근
- 다나와 모델명 기반 검색
- 제목/가격/상태/링크 추출
- 상세 페이지에서 추가 스펙/상태 보강

### 주요 blocker
- 요청 제한/차단
- 구조 변경 시 유지보수 비용 발생
- 검색 결과 품질이 광고/중복에 영향을 받을 수 있음

### 데이터 품질 이슈
- 판매량은 많지만 잡다한 키워드가 많이 섞일 수 있음
- 부품/완본체/주변기기 혼재 가능
- 상태값(판매중/예약중/판매완료) 파싱 규칙 필요

### 추천 우선순위
- **중상**
- 일반 대중 중고 마켓 중 구현 가치가 높음

---

## 5) 중고나라

### 접근 방식
- 웹/앱 구조 조사 필요
- 공개 검색이 제한적일 수 있음
- 일부 경로는 로그인/앱 의존 가능성 존재

### 수집 접근
- 공개 검색 페이지가 있으면 검색 기반 수집
- 없으면 feasibility 낮음
- 상세 페이지 접근 제약 여부 먼저 확인 필요

### 주요 blocker
- 로그인/앱 의존 가능성
- anti-bot 및 접근 제한 가능성
- 수집 안정성 낮을 수 있음

### 데이터 품질 이슈
- 게시글 수는 많아도 잡음도 큼
- 사기 방지 문구, 교환, 삽니다 글 많이 섞일 가능성
- 모델명 정확도는 소스별 편차 큼

### 추천 우선순위
- **중하**
- 볼륨은 크지만 안정적인 수집 가능성 확인이 먼저 필요

---

## 구현 우선순위 제안
1. **다나와 기준 상품 마스터 정교화**
2. **쿨앤조이 adapter**
3. **퀘이사존 adapter**
4. **번개장터 adapter**
5. **당근마켓 adapter**
6. **중고나라 adapter**

이 순서가 좋은 이유:
- 초반엔 모델명이 정확한 커뮤니티 장터가 매핑 정확도 개선에 유리
- 그 다음 거래량이 큰 일반 마켓으로 확장
- 가장 제약이 큰 소스는 뒤로 미룸

---

## 구현 시 주의
- 각 소스별 수집기는 반드시 독립 adapter 로 분리
- 소스 raw payload 보존 필드 마련 필요
- 판매완료 여부는 별도 정규화 필드 필요
- "삽니다", "교환", "고장", "부품용", "본체", "세트" 필터 규칙 필요
- GPU는 제조사와 칩셋(NVIDIA / ASUS / MSI / 3070 / Ti) 분리를 신경써야 함
- SSD/RAM은 용량/규격(DDR4/DDR5, NVMe/SATA) 토큰을 함께 관리해야 함
